## 2. 环境配置

### 2.1 双系统

算法组需要学会刷机的过程，这个过程和在自己电脑安装双系统差不多，所以笔者建议每位算法组的同学都安装一个**Ubuntu22.04**双系统，并且***尽量尝试将自己除了三维建模之外的所有工作都尝试搬到ubuntu上来做***。

[Windows 10 和 Ubuntu 双系统的安装和卸载](https://www.bilibili.com/video/BV1554y1n7zv/?share_source=copy_web&vd_source=00f98213d19a046d409fe94fb6c9f5fc)

~~并且可以尝试配置一个漂亮的自定义ubuntu界面~~

然后你就拥有了一个linux系统，需要学习一些基本的linux终端命令，好多都是边用边学，但下面这几个我觉得是最基本的。

- **Ctrl+Alt+T**在主目录打开新的终端
- 终端中**方向键上**可以找上一条你使用过的指令
- **cd**去到某个目录，此处建议大家学习一下[相对路径与绝对路径的概念](https://zhuanlan.zhihu.com/p/560426605)
- **ls**全名为list,列出当前目录下所有文件

***上述几条必须会。***

### 2.2 ROS2

使用下述指令安装

```bash
wget http://fishros.com/install -O fishros && . fishros
```

ROS（Robot Operating System，机器人操作系统），学习链接如下：

[【鱼香ROS】动手学ROS2|ROS2基础入门到实践教程|小鱼带你手把手学习ROS2](https://www.bilibili.com/video/BV1gr4y1Q7j5/?share_source=copy_web&vd_source=00f98213d19a046d409fe94fb6c9f5fc)

要了解最基本的指令，并学会使用C++实现一些基本功能，写个功能包之类的，在逐渐开始了解C++，cmake等概念，这些都是要边用边学的东西。

### 2.3 导航代码

[北极熊导航代码开源链接](https://github.com/SMBU-PolarBear-Robotics-Team/pb2025_sentry_nav.git)

我们的导航代码基于北极熊战队的开源进行开发。[仓库地址](https://github.com/LemperorD/nav2_ws.git)

关于如何跑通仿真，在我们的仓库里有非常非常详细的文档。

我们使用github管理代码，大家需要注册自己的github账号以及学习使用git这个工具，可以自己尝试开发一个C++项目（比如写一个EKF软件库）然后自己用一下git进行代码版本管理。

对于导航代码这部分的学习要求，最基本的要求就是把代码clone下来，配好环境跑通仿真。进一步地，大家要先从ros2的眼光理解工作空间和功能包是什么，然后按照上述的导航基本概念理解每个功能包是做什么的。

### 2.4 到底要学些什么呢

#### a.技术层面

技术层面，例如**C++,cmake,git,markdown**等等技术层面的知识，如果拿着文档死啃一定很难学，**建议大家边用边学，善用ai检查语法**。

然后建议大家也学习一下电控的部分，学习一下stm32怎么开发，与我们交流最多的组还是要互相了解。
